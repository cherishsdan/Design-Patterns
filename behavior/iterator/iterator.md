# iterator

> 迭代器模式是指提供一种方法访问一个聚合对象中的各个元素，而又不需要暴露该对象的内部表示
迭代器模式可以把迭代的过程从业务逻辑中分离出来。

实现一个迭代器

`
    function each(arr,callback){
        for( var i = 0, l = arr.length; i < l; i++){
            callback.call(arr[i],i,arr[i]);
        }
    };
    
    each([1,2,3],function(i,n){
        console.log(n);
    });

`

## 内部迭代器

上面的each函数就属于内部迭代器，each函数内部已经定义好了迭代规则，它完全接手整个迭代
过程，外部只需要一次初始调用。而它的缺点也在此，由于迭代器规则已经被定义好，each函数
不能迭代两个数组。

## 外部迭代器

外部迭代器必须显式的请求迭代下一个元素。虽然增加了迭代的复杂度，但相对也增强了迭代器的
灵活性，我们可以手工的控制迭代的顺序和过程。
